# Deployment

## Full-Stack Platform: Render Blueprint

Source repo:

https://github.com/miniHaLe/phongthanh-admin

Blueprint file:

```text
render.yaml
```

Deploy from:

https://render.com/deploy?repo=https://github.com/miniHaLe/phongthanh-admin

The Blueprint provisions:

- `phongthanh-admin-minihale` — static React/Vite frontend
- `phongthanh-admin-api-minihale` — NestJS API
- `phongthanh-admin-db-minihale` — Postgres database

Expected URLs:

- Frontend: `https://phongthanh-admin-minihale.onrender.com`
- API: `https://phongthanh-admin-api-minihale.onrender.com`
- API health: `https://phongthanh-admin-api-minihale.onrender.com/health`

## Required Render Input

Render prompts for one secret during Blueprint creation:

- `INITIAL_ADMIN_PASSWORD` — initial seeded password for `admin` and seeded users.

Do not commit this value. The seed forces users to change password on first login.

## Environment Variables

Frontend:

- `VITE_REAL_RESOURCES=khach-hang`
- `VITE_API_URL=https://phongthanh-admin-api-minihale.onrender.com`

API:

- `DATABASE_URL` — injected from the Render Postgres database
- `JWT_SECRET` — generated by Render
- `JWT_REFRESH_SECRET` — generated by Render
- `INITIAL_ADMIN_PASSWORD` — entered during Blueprint creation
- `CORS_ORIGIN=https://phongthanh-admin-minihale.onrender.com,https://minihale.github.io`
- `REFRESH_COOKIE_SAME_SITE=none`

## Build And Start

Frontend:

```bash
npm ci && npm run build:prod
```

API:

```bash
npm ci && npm run build
npm run db:migrate && npm run seed
npm run start:prod
```

## Existing Static Preview

GitHub Pages frontend:

https://minihale.github.io/

This is also rebuilt against
`https://phongthanh-admin-api-minihale.onrender.com`; it will work once the
Render API/database services are provisioned. The Render frontend remains the
intended single-platform full-stack deployment.

GitHub Pages cannot rewrite clean SPA paths to `index.html` with a 200 status.
The app therefore switches to hash routing on `*.github.io`, and `public/404.html`
redirects old clean URLs such as `/trang-chu` to `/#/trang-chu`.

## Rollback

Use Render Dashboard -> service -> Events -> previous deploy -> Manual Deploy.

## Notes

Free Render web services can sleep after inactivity, causing a cold start on the
first request. Free Postgres is suitable for demo/staging; check the current
Render plan limits before relying on it for long-lived production data.
