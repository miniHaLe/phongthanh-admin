## Risk Assessment

- **15-file status migration breaks rendering silently** (highest risk): mitigated by characterization tests written first (step 3) against the LIVE layers + numeric-union type forces compile errors at every consumer — no stringly fallthrough. **C1 correction:** `dashboard-mock.ts` DOES import `status.ts` (it's one of the 15), so the compiler flags it — the only layer needing hand-migration for vocab is `sua-chua-report-mock.ts` (its own `REPAIR_STATUSES_VN`, no status.ts import). `dashboard-mock.ts` still needs the non-type reworks (bucket→OPEN_STATUS_IDS, seeded isOverdue) which the compiler won't flag — those are explicit in step 7.
- **URL filter params change shape** (snake slugs → numeric ids): saved views / bookmarks in localStorage (`pt-saved-views`, `pt-filter-state`) hold stale slugs. Mitigation: single `parseStatusIds()` on every restore path (drop non-numeric/unknown → fall back to `OPEN_STATUS_IDS`, Finding 5); mock app, `resetDemo()` available.
- **Lookup shape churn ripples into P5/P6**: mitigated by locking lookup id/label pairs with deep-equal tests — later phases import, never redefine (plan.md ownership rule).
- **Wrong layer edited (D5 regression)**: the risk this whole reconciliation removes — edit the layer a page *reads* (`MOCK_TICKETS`/`dashboard-mock.ts`), not the dead `SEED_*` arrays. The step-20 grep for `SEED_REPAIR_TICKETS`/`SEED_CUSTOMERS`/`SEED_FINANCIALS` returning nothing is the proof the dead arrays are gone; the characterization suite runs against the live layer so green means rendered data actually changed.
- **Hex contrast (e.g. #FFCC00, #00FF00) fails WCAG with white text**: P1 badge design uses hex swatch + foreground-colored label, deferring the reference white-pill-on-hex-td render to P3 where the exact legacy markup is replicated.
- **Vitest/Vite 8 compatibility**: use current vitest major supporting Vite 8; if the `test` block conflicts, split into `vitest.config.ts` merging vite config — contained to step 2.
- **Rollback**: phase is one commit chain touching no server/state contracts; `git revert` of the phase commits restores the 16-status app; no data migration to unwind (all mock/deterministic).
